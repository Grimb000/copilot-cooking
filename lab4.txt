 Твоя задача — выполнить первую часть лабораторной работы по тестированию Android-приложений (Задания 1–7) и сохранить все артефакты в текущей директории проекта. Это уже 4 лабораторная работа и файлы для нее также создаются отдельно от предыдущих лабораторных работ не изменяя прошлые лабораторные работы.

У нас есть два базовых Android-приложения, которые нужно покрыть тестами: 
1. "Калькулятор" 
2. "Угадай число" 

Выполни следующие задачи строго по шагам. Пиши полный, рабочий код на Kotlin (или Java) и создавай необходимые файлы.

### Шаг 1: Подготовка отчета (Задание 1)
Создай файл `docs/report.md` (или обнови его, если он есть) и добавь туда раздел "Задание 1. Виды тестов в Android Studio". В этом разделе напиши ответы на следующие теоретические вопросы:

### Шаг 2: Тестирование "Калькулятора" (Задания 2, 3 и 4)
Создай необходимые директории и сгенерируй файлы с тестами:

**Задание 2. Модульное тестирование:**
Создай файл `calculator/src/test/java/com/example/calculator/CalculatorUnitTest.kt`.
Используя JUnit4, напиши минимум 5 тестов, проверяющих математику без запуска интерфейса:
- `testAddition_Positive()`: 2 + 3 = 5
- `testDivision_ByZero()`: Проверка выброса исключения `ArithmeticException` при делении на ноль (используй `assertThrows`).
- `testSubtraction_NegativeResult()`: 3 - 5 = -2
- `testMultiplication_WithZero()`: 10 * 0 = 0
- И добавь еще один тест на твое усмотрение.

**Задание 3. Инструментальное тестирование UI с Espresso:**
Создай файл `calculator/src/androidTest/java/com/example/calculator/CalculatorUITest.kt`.
Используя Espresso, напиши минимум 5 UI-тестов, имитирующих нажатия на кнопки и проверку `TextView` с результатом. Примеры тестов:
- `testUi_SimpleAddition()`: нажать "2", "+", "3", "=", проверить результат "5".
- `testUi_ChainOperations()`: ввод "5 + 5 - 2", проверка результата "8".
- `testUi_DivisionByZero_ShowsError()`: ввод "10 / 0", нажать "=", проверить отображение текста "Error" или "Infinity".

**Задание 4. Тестирование сохранения состояния при повороте экрана:**
Добавь в класс `CalculatorUITest` (из Задания 3) тест `testUi_RotationSavesInput()`.
Используй `ActivityScenario.recreate()` для управления жизненным циклом. 
Сценарий: набрать на калькуляторе "123", вызвать `recreate()` (имитация поворота), проверить, что в поле вывода все еще отображается "123".

В файл `docs/report.md` добавь разделы "Задание 2", "Задание 3" и "Задание 4" с кратким описанием написанных тестов.

### Шаг 3: Тестирование "Угадай число" (Задания 5, 6 и 7)
Создай директории и файлы для приложения:

**Задание 5. Модульное тестирование игровой логики:**
Создай файл `guessnumber/src/test/java/com/example/guessnumber/GameUnitTest.kt`.
Предположим, что интерфейс генератора случайных чисел называется `NumberGenerator`. Используя библиотеку `Mockito`, подмени генератор, заставив его возвращать число 42.
Напиши минимум 5 тестов (включая следующие):
- `testGuess_Lower()`: при вызове `guess(30)` результат должен быть "Больше".
- `testGuess_Higher()`: при вызове `guess(50)` результат должен быть "Меньше".
- `testGuess_Correct()`: при вызове `guess(42)` результат равен "Угадал" и метод `isGameOver()` возвращает `true`.

**Задание 6. Тестирование полного сценария пользователя:**
Создай файл `guessnumber/src/androidTest/java/com/example/guessnumber/GameUITest.kt`.
Напиши UI-тест `testFullGameCycle()`, имитирующий полный игровой цикл с использованием Espresso:
- Ввести число 50, нажать "Проверить".
- Проверить появление текста "Меньше" или "Больше".
- Ввести правильное число (считаем, что оно известно или мокается).
- Проверить появление сообщения о победе и кнопки "Новая игра".
Напиши еще один тест `testInputValidation()`, который вводит текст "abc", нажимает "Проверить" и проверяет отображение ошибки (например, Toast или подсветку поля красным).

**Задание 7. Расширенное UI-тестирование:**
В класс `GameUITest` добавь минимум 2 расширенных теста:
1. `testAutoKeyboardHiding()`: Кликнуть на поле ввода (появится клавиатура), нажать кнопку "Проверить", использовать метод `isNotDisplayed()` для проверки того, что программная клавиатура была скрыта с экрана.
2. `testAccessibility()`: Добавить тест доступности для незрячих пользователей. Использовать `AccessibilityChecks` из Espresso для автоматического сканирования экрана на предмет проблем, а также явно проверить наличие атрибута `contentDescription` у кнопок "Проверить" и "Новая игра".

В файл `docs/report.md` добавь разделы "Задание 5", "Задание 6" и "Задание 7", кратко описав эти тесты. Также по каждому заданию в этом файле как-то напиши откуда мне и какой скрин сделать для того что показать что я сделал по каждому заданию.

### Финализация:
Убедись, что:
1. Код содержит все необходимые `import` (включая JUnit, Espresso, Mockito).
2. Файлы созданы в правильной структуре: `src/test/java/` для модульных и `src/androidTest/java/` для UI-тестов.
3. Отчет `docs/report.md` заполнен ответами и описаниями тестов.
